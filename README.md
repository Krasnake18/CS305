For Artemis Financial, a financial services company, who wanted to improve the security of the web application that they have that handles sensitive client data.  The biggest issue that I identified and fixed were software vulnerabilities that could lead to the exposure of financial information.  I used a vulnerability assessment using tools including OWASP Dependency-Check to identify risks that were in third-party libraries and the application code.  I also used secure coding practices like enforcing HTTPS with TLS for secure communication and by using SHA-256 hashing to verify the data’s integrity.  By using secure coding, we can improve the protection of sensitive data, decrease risk of breaches, doing so will help maintain customer trust.
 One of my most challenging aspects was determining which of the vulnerabilities were actual risks versus false positives.  Though this did give me a greater insight into how to interpret security reports.  I used layers of security through encryption, hashing, and dependency management.  Functionality was tested by running the application once refactoring was completed to make sure everything worked correctly.  After doing so I completed a second dependency check to make sure that no new vulnerabilities were added.  This project is able to demonstrate my ability to analyze security issues, use mitigation strategies, and my use of tools such as Maven and OWASP Dependency Check.
